JavaScript A Underdog

From last few decades, we saw some great programming languages C++ and JAVA being primary and most influential among others, we also have some really good and light weight programming languages server side development too (primarily php). But when it comes to client side scripting nothing is better than JavaScript.

And probably this is the biggest problem with JavaScript, that most of the people think it is just a small client side scripting language, and does not have what it takes to consider it a legitimate language that can handle heavy programming, but that is wrong and let's see why.

History

Javascript was developed in May 1995 by Brendan Each in just ten days. The original name was Mocha. In September of 1995 the name was changed to LiveScript, then in December of the same year, on receiving a trademark license from Sun, the name JavaScript was adapted.

For JavaScript to be used by every vendor it needed standards to maintain. In 1996 - 1997 JavaScript was taken to ECMA (European Computer Manufacturers Association) to carve out a standard specification, which other browser vendors could then implement based on the work done at Netscape. ECMAScript is the name of the official standard, with JavaScript being the most well known of the implementations. ActionScript 3 is another well-known implementation of ECMAScript, with extensions. Although ECMA released ES6, a lot of browsers still working on ES5.

Programming

Learning JavaScript is much easier than it looks, if you have some understanding of programming, even a little is enough. The best thing about JavaScript is it provide complete freedom to the developers. It is not strictly typed language, so we can create a variable and simply initialize with any type and later we can store some other type of variable too.
And not only that, we can choose process oriented programming (like c programming), Object oriented programming, or functional programming.

Object Orientation

Object Oriented Programming (OOP) refers to using self-contained pieces of code to develop applications.We use objects as building blocks for our applications. Building applications with objects allows us to adapt some valuable techniques, namely Inheritance,Polymorphism , and Encapsulation. since only Inheritance and Encapsulation concepts apply to OOP in JavaScript, particularly because, in JavaScript, objects can encapsulate functionalities and inherit methods.

Encapsulation refers to enclosing all the functionalities of an object within that object so that the objects internal workings (its methods and properties) are hidden from the rest of the application. This allows us to abstract or localize specific set of functionalities on objects. It is archived using function constructor in JavaScript. 

function Employee (firstName, lastName, startDate, signedNDA) {
	this.firstName = firstName;
	this.lastName = lastName;
	this.startDate = startDate;
	this.signedNDA = signedNDA;
	this.fullName = function () {
		console.log (this.firstName + " " + this.lastName); 
	};

}
var abhijit = new Employee ("Rahul", "Kant", new Date(), true);

We can do the same thing using class and constructor.

Inheritance refers to an object being able to inherit methods and properties from a parent object (a Class in other OOP languages, or a Function in JavaScript). This is archived by prototype concept in JavaScript. 

function Employee () {}

Employee.prototype.firstName = "Abhijit";

Employee.prototype.startDate = new Date();
Employee.prototype.signedNDA = false;

Functional Programming

According to an article from Eric Elliott "Functional programming (often abbreviated FP) is the process of building software by composing pure functions, avoiding shared state, mutable data, and side-effects. Functional programming is declarative rather than imperative, and application state flows through pure functions. Contrast with object oriented programming, where application state is usually shared and collocated with methods in objects.". It is easy and really interesting, I will really recommend to read it.

Other Technologies

Now a days you can do a lot more than just front end scripting on JavaScript. JavaScript is the main reason that we have noSQL databases like mongoDB and couchDB. We have a technology called Nodejs which is mainly concentrated to serve web requests, it is lighter and faster than most other application server technologies. Then there some technologies that let you to create desktop application using JavaScript (appjs, tangiblejs, electron are most popular).
